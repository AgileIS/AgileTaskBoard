Layout & Configuration

Below is a brief description of the layout and settings of the agile task board.

Layout

The layout consists of the header area, which contains the title, the description and the header bar. This is followed by the task panel, which is used to manage the tasks.

The header bar contains configured users and tags that can be assigned to a task via drag & drop. A click on a user or a tag filters the tasks. Another click cancels the filter again. With the magnifying glass you can filter by title.

Clicking on a task title opens the edit panel. Here you can edit properties, link documents or add a comment.

The behavior of the agile task board can be adjusted via the settings panel.

Settings

Basic​

Title​

A title for the task board can be assigned here.

Description​

Here you can add a description to the task board.

Task list​

The task list (SharePoint list) where the tasks are stored. You can create a new list using the "Create" button.

Multiple board instances can work with one list. This allows different views of the tasks. The templates Scrum - Planning Board and Scrum - Sprint Board are a good example of this.

Status​

The status field controls the visibility of the status columns. The "Edit Options" button allows you to add, adjust or remove status entries.

People​

Here you define the users that should be visible in the header bar. These can be assigned by drag & drop and filtered by click.

Preferred Tags​

Here you define the tags that are visible in the header. Tags can be assigned a color. Tasks are colored based on the color of the tag. Drag & drop assignment and filtering apply to tags just as they do to users.

'Not assigned' swimlane name​

The name for the default swimlane. It is used when a task is not assigned to a swimlane or no swimlane has been set.

Swimlanes​

This field defines the swimlanes of a board.

A swimlane can be used to classify tasks on a board.

Enhanced Features​

Checklist​

Activates the Checklist field. With the help of the checklist a simple form of subtasks can be realized.

Use Planner​

Enables the use of Microsoft Planner to create and link subtasks.

Doing & Done​

Allows to divide a status column into doing and done.

Doing and done column selection​

Here you can select the status columns for which doing&done should be enabled.

Work in Progress​

Allows the specification of a maximum number of tasks per status. If the number is exceeded, a red number is displayed. This option only includes the visual representation, there is no functionality to prevent exceeding the number.

Attached links​

Allows to create and display links within the attachment tab of the edit panel. If multiple links are specified, one can be used as the primary one. The primary link can be used as the subtitle field of the task card.

Documents​

Enables the possibility to link documents to a task.

Linking of documents is possible via upload and selection of already existing documents.

Document library​

The document library into which the linked documents are uploaded or from which documents can be selected.

This option is set in teams fix to the documents library Shared Documents

Allow folder​

Allows the creation of folders

Comments​

Allows the creation of comments, for this the comments tab is added to the edit panel.

With "@mention" a person can be mentioned. This person will receive a notification when the appropriate notification is activated.

History​

Enables the version history of the SharePoint list and adds a history tab to the edit panel.

Enable 'live updates'​

Allows to receive changes without reloading the board.

Appearance​

Customize card​

The task and the edit panel can be customized via this panel.

Card​

Allows you to customize the fields displayed on the task card. It is also possible to customize the behavior when clicking.

Edit panel​

Here you can define which fields should be displayed in which order on the properties tab.

Card sorting​

Sets the sort order in which the tasks are displayed.

Last column filter​

This filter is applied only to the last column of the panel. It can be used, for example, to not display old tasks. The where part of a CAML query is used as filter definition.

Default value:

<Geq><FieldRef Name='Modified' /><Value Type='DateTime'><Today OffsetDays='-30' /></Value></Geq>

Query schema

Fixed header bar​

If enabled, the header bar is fixed at the top of the window when scrolling.

Card full with​

A task always takes up the entire width of a column.

Collapse swimlanes​

All swimlanes are displayed minimized when opening the board.

Collapse empty swimlanes​

Empty swimlanes are displayed minimized when opening the board.

Show due date always​

If enabled, a due date icon is always displayed on the task. Otherwise, the due date icon is only displayed when the warning offset is reached or the date is in the past.

ColorDescription
greenThe due date is greater than the offset in the future
yellowThe due date is lower than the offset
redThe due date is in the past

Due date warning offset​

This option specifies the due date warning offset in days.

Notifications​

OptionDescription
On mentionNotify a user when mentioned inside a comment
On assignmentNotify a user when a task is assigned to him
On updateNotify a user when a task assigned to him was modified

Diagnostics​

Allows the transmission of error reports and performance information for the purpose of optimizing the product.
